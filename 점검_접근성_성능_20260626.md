# 핑(Ping) 접근성·성능 점검 (2026-06-26)

대상: `app.html` · 기준: WCAG 2.1/2.2 AA

---

## 한 줄 결론
성능은 양호. 접근성은 **색상 대비(가독성)**가 가장 큰 문제 — 보조 텍스트 색이 AA 기준(4.5:1)에 미달. 대부분 CSS 변수 몇 줄 수정으로 해결 가능.

---

## 접근성 (우선순위 순)

### 🔴 1. 색상 대비 미달 (WCAG 1.4.3) — 핵심
흰색/크림 배경 기준 실제 대비 계산값:

| 색 변수 | 값 | 대비 | 판정 | 쓰이는 곳(예) |
|---|---|---|---|---|
| `--gray500` | #6B6159 | 6.0:1 | ✅ 통과 | 본문 보조 |
| `--gray400` | #9E9189 | **3.1:1** | ❌ 미달 | 부제목·미리보기·캡션·시간 |
| `--gray300` | #C4B8AC | **1.9:1** | ❌ 심각 | 하단 탭(비활성)·시간·플레이스홀더 |
| `--orange` 텍스트 | #FF6B35 | **2.8:1** | ❌ 미달 | 인사말·진행도 등 작은 글자 |

영향: 부제목, 카드 시간, 댓글 미리보기, **하단 네비게이션의 비활성 탭** 등 회색 작은 글씨가 잘 안 보임. 특히 `gray300`(1.9:1)은 거의 안 읽힘.

**수정안 (CSS `:root` 한 곳):**
- `--gray400: #9E9189` → `#786C64` (약 4.6:1)
- `--gray300`은 **텍스트로 쓰지 말 것**. 텍스트엔 `gray400/500` 사용, 테두리·구분선·배경에만 `gray300`.
- 주황색(`#FF6B35`)은 작은 본문 텍스트로 쓰지 말고, **큰 글자/굵은 글자**나 흰 글씨+주황 배경에만. 작은 라벨은 `--navy(#004E89)`나 `gray500`로.

### 🟡 2. 입력 요소 이름 누락 (WCAG 3.3.2 / 4.1.2)
- `#zoomSlider`(줌 슬라이더): 라벨/`aria-label` 없음 → 스크린리더가 "슬라이더"로만 읽음. → `aria-label="카메라 줌"` 추가.
- `#commentInput`·`#commentText`: placeholder만 있음(placeholder는 라벨 대체 불가). → `aria-label` 추가 권장.
- ✅ 잘된 점: 산책·가족모임 체크박스는 `<label>`로 감싸 텍스트 라벨 있음.

### 🟡 3. 키보드 포커스 표시 없음 (WCAG 2.4.7)
- `:focus`/`:focus-visible` 스타일이 정의돼 있지 않고 `-webkit-tap-highlight-color:transparent`로 기본 강조도 제거됨. 터치엔 `:active`가 있어 괜찮지만, 키보드/접근성 기기 사용자는 포커스 위치를 알 수 없음.
- → 전역 `:focus-visible{outline:2px solid var(--navy);outline-offset:2px}` 추가 권장.

### 🟢 4. 터치 영역 (WCAG 2.2 기준 대체로 통과)
- WCAG 2.2 AA 최소(24×24px)는 대부분 충족. 단 `reaction-chip`(공감 칩), `own-actions`/`day-item` 버튼, `ib-close`(26×26)는 작은 편.
- 권장(AAA 44×44는 아니어도): 자주 누르는 버튼은 `min-height` 키우면 가족(특히 어르신) 사용성↑.

### 🟢 5. 이미지 대체텍스트 — 양호
- 피드 사진 `alt="OO의 핑"`, 아이콘 `alt`에 이모지 폴백까지 잘 됨.
- 앨범·달력 썸네일은 `alt=""`(장식 취급)인데 옆에 이름/날짜 텍스트가 있어 허용 범위. 굳이 고칠 필요 낮음.

## 성능 — 전반적으로 양호

### 잘 된 점
- 단일 HTML(약 176KB) 인라인 → 요청 1번. 서비스워커(network-first) 캐시. PWA로 적절.
- Firestore 실시간 리스너 4개(멤버별)뿐 — 가볍다.

### 소소한 개선 여지
1. **카운트다운 초당 재생성** (`updateCountdown`, 1초 간격): 매 초 `innerHTML` 전체를 다시 쓰고, 그 안에 `eico()`가 `<img>` 태그를 매번 새로 만듦. 동작엔 문제없지만 1초마다 DOM 노드 재생성. → 라벨/아이콘은 한 번만 만들고 **시:분:초 숫자만 갱신**하면 더 매끈. (핸드오프의 "카운트다운이 버벅"과도 연결.)
2. **앨범 12개월 전체 조회**: 앨범 열 때마다 `albumCache=null`로 초기화 후 최근 12개월 문서를 매번 `getDocs`. 가족 4인이라 양은 적지만, 1년 누적되면 수백 건. 지금은 OK, 나중에 월 단위 지연 로딩 고려.
