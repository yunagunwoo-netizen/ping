# 핑 서비스 (v2) — 반려가족 맞춤 플랫폼 프로젝트

> 우리 가족용 라이브 앱(`핑` 폴더 최상위의 `app.html` 등)과 **분리된** 서비스화 프로젝트 폴더입니다.
> 이 폴더만 다른 위치로 옮겨도 독립적으로 동작합니다.

## 폴더 구성

| 파일 | 설명 |
|---|---|
| `핑_서비스기획서.docx` | 서비스 기획서(시장·기능·BM·로드맵 등) |
| `데이터구조_설계.md` | 멀티테넌시 Firestore 구조·보안 설계 |
| `onboarding-prototype.html` | 온보딩 UI 프로토타입(저장 안 함, 데모) |
| `onboarding-v2.html` | 온보딩 — 실제 Firebase 저장(가족·펫·보상 생성) |
| `app-v2.html` | v2 앱 본체 — 로그인→가족 로드→데일리 핑→목표→초대 |
| `lib-ping-v2.js` | 인증 + 가족 컨텍스트 + 데이터 계층(공용 모듈) |
| `firestore.rules` | 멀티테넌시 Firestore 보안 규칙 |
| `storage.rules` | 멀티테넌시 Storage 보안 규칙 |
| (이미지) | 로고·마스코트·아바타 등 데모용 자산 |

## 테스트 방법

1. 이 폴더를 웹에 올리거나(예: 별도 GitHub Pages/도메인), 로컬 서버로 실행.
   - ⚠️ Firebase 인증은 **승인된 도메인**에서만 동작 → `file://`로 직접 열면 막힘. localhost 또는 배포 URL 사용.
2. `onboarding-v2.html` 접속 → "새 가족 만들기" → 펫·보상·점수 설정 → 완료 시 **초대코드** 발급.
3. 같은 브라우저로 `app-v2.html` 접속 → 그 가족이 로드됨 → 데일리 핑/목표/초대 동작.

## 주의

- 현재 데모는 기존 운영 프로젝트(`ping-family`)의 **새 `families/` 컬렉션**에 저장됩니다(라이브 가족 앱의 `pings/`와 분리). 정식 서비스화 시 **별도 Firebase 프로젝트** 생성을 권장합니다.
- `firestore.rules`/`storage.rules`는 멀티테넌시 전용이라, **라이브 가족 앱 프로젝트에는 적용하지 마세요**(앱이 막힘).
